**Implementation Spec**

Build a Nuxt + Convex app where users pick one language, judge 10 code snippets as Human or AI, get snarky non-revealing feedback after each answer, then see a shareable result page with score, answer reveal, aggregate stats, and provenance links.

**Product Scope**
- Name/domain deferred.
- Tone: playful/snarky, never mean to the user.
- One language per run.
- Launch languages: JS, TS, Python, Rust, C++.
- Quiz length: 10 mandatory questions.
- Labels per run: 4-6 Human and 4-6 AI.
- Retakes allowed with a new randomized run.
- Avoid recently seen pairs via local storage when possible.
- No login.
- Result pages are unlisted but shareable via short slug.
- No leaderboard for v1.
- X share includes score, language, link, and hashtag.

**Quiz Flow**
1. Landing page shows title, hook, language picker, start button, methodology link, and “10 real code snippets. No login.”
2. Optional demographic screen appears after language selection if `ENABLE_DEMOGRAPHIC_PROMPTS` is enabled.
3. Server creates session after the prompt screen.
4. Client receives 10 display-only questions.
5. User answers each with `Human` or `AI`.
6. UI immediately locks answer and shows a random neutral snark interstitial for 600-900ms.
7. Answer writes to Convex in background with retry.
8. Correctness is not revealed during quiz.
9. Results reveal score, correct labels, user guesses, stats, and provenance.
10. Public shared result page does not show full snippets by default.

**Snippet Rules**
- Prefer standalone functions for v1.
- Classes allowed later only if short/self-contained.
- Target length: 15-45 display lines.
- Hard cap: about 70 lines.
- Reject below ~8 meaningful lines unless unusually interesting.
- Strip comments/docstrings.
- Format Human and AI with same formatter.
- Preserve local variable names.
- Anonymize top-level function/class names and project-specific identifiers.
- Keep type annotations/signatures as-is after anonymization.
- Allow obvious helper calls only if not core hidden logic.
- Hide repo/file/source until results.
- Show syntax highlighting and subtle line numbers.
- No copy button during quiz.

**Formatters**
- JS/TS: Prettier.
- Python: Black.
- Rust: rustfmt.
- C++: clang-format with fixed style.

**Dataset Target**
- Launch target: 40 pairs per language.
- Total launch dataset: 200 pairs / 400 snippets.
- Each pair has one Human implementation and one AI implementation for the same behavioral task.
- Public dataset lives in repo under `dataset/v1/`.
- Use mock/dev dataset first with the real schema.

**Human Snippet Sourcing**
- Human code comes from permissively licensed open-source repos at pre-2021 refs.
- Use curated `repos.yml`, not open-ended GitHub search.
- Clone repos locally at pinned refs.
- Prefer credible but not meme-famous repos.
- Avoid instantly recognizable codebases/functions.
- Use exact commit SHA and source line anchors.
- Exclude GPL/AGPL/LGPL/MPL/EPL for v1.
- Allowed licenses: MIT, Apache-2.0, BSD-2-Clause, BSD-3-Clause, ISC, Zlib, Boost.
- Generate `ATTRIBUTIONS.md`.

**Dataset Pipeline**
Place scripts inside the Nuxt repo under `scripts/dataset/`.

Commands:
- `collect`: clone/read configured repos and extract candidate functions.
- `filter`: apply length/path/context/license heuristics.
- `spec`: generate structured behavioral specs from normalized/anonymized human snippets.
- `generate`: generate AI implementations from specs using GPT-5.5.
- `normalize`: strip comments, anonymize names, format snippets.
- `validate`: parse/format/schema validation.
- `review`: generate local JSON/HTML review artifact for accept/reject.
- `export`: write final JSONL dataset.
- `import`: load approved dataset into Convex.

Parsing:
- Use tree-sitter for JS, TS, Python, Rust, C++.
- Use JSON Schema validation for generated specs and dataset records.

**AI Generation Policy**
- UI label: “Human or AI?”
- Dataset discloses AI snippets generated by GPT-5.5.
- Same model family can be used for spec and code generation, but with separate prompt versions/calls.
- Spec is generated from normalized/anonymized human display version plus language/signature metadata.
- AI generation tries to match language, signature shape, input/output shape, sync/async nature, and type richness.
- Do not force same variable names or exact algorithm.
- Automated formatting and bounded retries allowed.
- Retry up to 2 times for parse/format failures.
- Human review is accept/reject only.
- No manual editing of AI code.
- Publish approved pairs only.

**Spec Schema**
Publish full behavioral spec plus prompt/model metadata.

Include:
- `schema_version`
- `summary`
- `signature_intent`
- `behavior`
- `edge_cases`
- `forbidden_details`
- `spec.model`
- `spec.generated_at`
- `ai.generation_prompt_version`
- `ai.model`
- `ai.generated_at`

Reject specs that leak original variable names, project names, source repo, or unnecessary algorithmic quirks.

**Convex Data Model**
Core tables:
- `pairs`: imported approved dataset records.
- `sessions`: language, result slug, profile answers, status, dataset version, timestamps.
- `sessionQuestions`: session assignment, order, pair reference, shown label.
- `answers`: immutable guesses, correctness, response time.
- `aggregates`: counters by language, pair, label, score distribution, optional segments.

Session behavior:
- Server creates `resultSlug` at session start.
- Public result route shows “quiz not completed yet” if incomplete.
- Completed result pages remain shareable indefinitely.
- Incomplete reload resume allowed within 5 minutes.
- No hard active in-page timer.
- Server can clean up absurdly old incomplete sessions later, e.g. 24h.

**Quiz Generation**
On demand:
1. Filter approved pairs by selected language.
2. Exclude local-storage recent pair IDs if enough remain.
3. Randomly sample 10 distinct pairs server-side.
4. Assign shown labels with each class count between 4 and 6.
5. Store session and questions in Convex.
6. Return display-only payload.

Client payload includes:
- `sessionId`
- `resultSlug`
- `language`
- `questions`: `questionId`, `displayCode`, `displayLanguage`, maybe `lineCount`
- `featureFlags`
- `expiresAt` for resume only

Client payload excludes:
- Correct label
- Source URL
- AI model/spec
- Aggregate stats
- Pair ID if avoidable

**Answer Handling**
- User cannot skip questions.
- User cannot go back/change answers.
- Client disables buttons immediately after click.
- Client writes answer in background.
- Mutation validates session/question ownership.
- `(sessionId, questionId)` is idempotent.
- Same duplicate guess returns success.
- Different duplicate guess is rejected; first answer wins.
- Correctness computed and stored at submit time.
- Results page waits for pending answers to flush.

**Aggregates**
On answer submit:
- Increment pair shown count by `pair_id` + `shown_label`.
- Increment Human/AI guess counts.
- Increment correct count.
- Add response time.

On session completion:
- Increment completed sessions by language.
- Add score by language.
- Update score histogram `0..10`.
- Update optional demographic aggregates if enabled.

Show:
- User score.
- Average score by language.
- Total completed sessions.
- Percentile only after threshold, e.g. 100 completed sessions.
- Per-answer crowd stat after enough guesses, e.g. 20.

Do not use incomplete sessions for score/percentiles. Submitted answers from incomplete sessions can count for per-question guess distributions.

**Feature Flags**
- `ENABLE_DEMOGRAPHIC_PROMPTS`
- `ENABLE_PAIR_REPORTING`
- `ENABLED_LANGUAGES`
- `ENABLE_PUBLIC_AGGREGATES`
- `ENABLE_PERCENTILES`
- `ENABLE_SHARE_CARD`

Evaluate server-side for data/payload behavior. Send relevant flags to client for UI rendering.

**Demographics**
Optional and feature-flagged.

Questions:
- “Which best describes you?” professional / hobbyist-student / don’t code regularly / prefer not to say
- “How often do you use AI coding tools?” daily / sometimes / rarely-never / prefer not to say

Store at session level. Do not show individually on public result pages.

**Reporting**
Feature-flagged via `ENABLE_PAIR_REPORTING`.
- Result answer cards can show “Report issue.”
- Fastest v1: prefilled GitHub issue with pair ID, language, result slug, and checklist.
- Later: store feedback in Convex.

**Results Page**
Private immediate and public shared view can be the same compact result for v1.

Show:
- Score.
- Language.
- Completed time.
- Share CTA.
- Aggregate summary.
- Compact answer cards with user guess, correct label, correctness, crowd stat, source/provenance link, license/model metadata.
- Note if a pair was later deprecated: “This item was later removed from active rotation.”

Do not show all full snippets by default on public results.

**Methodology Page**
Add `/methodology`.

Sections:
- Game/informal experiment.
- Human snippets: permissive OSS, pre-2021 refs.
- AI snippets: GPT-5.5 generated from behavioral specs.
- Normalization: comments stripped, names anonymized, same formatter.
- Review: syntax validated, accept/reject only.
- Limitations: not an AI detector, not peer-reviewed, selection bias.
- Dataset link.

Mention pre-2021 on methodology and result source details, not landing hero.
